1. System Overview

ระบบยืนยันตัวตนผ่านบัตรประชาชนบนตู้ Kiosk แบบ Self-Service โดยผู้ใช้สอดบัตรประชาชน (Smart Card) สแกนใบหน้า และยืนยันข้อมูลด้วยตนเอง พร้อมรองรับ Consent Form ตาม NDPP (National Data Privacy Protection)

Main Flow

Step1 → Step2 → Step3 → Step4 → Step5 → Step8 → Step9 → Step10 → Step11 → Step12 → [Step14] → [Step17] → [Step18] → StepEnd

Step14, Step17, Step18 เป็น Conditional Steps จะแสดงเฉพาะเมื่อมี config/data ที่เกี่ยวข้อง ถ้าไม่มีจะข้ามไปอัตโนมัติ

2. Actors

ActorDescription
Visitor (ผู้ใช้บริการ)ยืนยันตัวตนที่หน้าตู้ Kiosk
Kiosk Terminalแสดง UI, สั่ง Card Reader, ถ่ายภาพใบหน้า
Card Reader (Hardware)อ่านข้อมูลจากชิปบัตรประชาชน ผ่าน SignalR
Camera (Hardware)ถ่ายภาพใบหน้าผู้ใช้
Web Server (N-Kiosk-Web)ประมวลผล, จัดเก็บข้อมูล, เรียก API ภายนอก (ASP.NET Core)
File Server (nginx)จัดเก็บไฟล์รูปภาพ (WebDAV PUT)
NDPP Server (External)รับ consent form — JWT Auth + REST API

3. Preconditions

  • ตู้ Kiosk เปิดอยู่และเชื่อมต่อ Web Server ได้
  • Card Reader เชื่อมต่อผ่าน SignalR Hub (CardReaderHub) สำเร็จ
  • กล้องพร้อมใช้งาน (browser permission granted)
  • ผู้ใช้มีบัตรประชาชนแบบ Smart Card ที่ยังไม่หมดอายุ
  • File Server (nginx) พร้อมรับ upload

9. Security

MeasureDescription
Payload Encryptionทุก request/response เข้ารหัส AES-ECB ระหว่าง Browser ↔ Web Server
JWT AuthenticationNDPP consent ใช้ JWT token sign ด้วย SSO_Key (stored in DB Variables)
Data Maskingข้อมูลอ่อนไหว (เลขบัตร) แสดงแบบ mask บน UI — unmask ได้ชั่วคราว
Session IsolationClear session storage ทุกครั้งที่เริ่มรอบใหม่ (Step1)
File Server ACLnginx allow/deny จำกัด subnet ที่ upload ได้
HTTPSProduction environment ใช้ HTTPS สำหรับ communication ทุกช่องทาง
