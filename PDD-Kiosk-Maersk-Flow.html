1. System Overview

ระบบลงทะเบียนและจัดการความยินยอม (Consent Management) ผ่านตู้ Kiosk แบบ Self-Service สำหรับ Maersk โดยผู้ใช้สอดบัตรประชาชน ถ่ายภาพใบหน้า และให้ความยินยอม (Consent) ตามกฎหมาย PDPA ก่อนส่งข้อมูลไปยังระบบ Netka EnTelligence

Main Flow (New User)

Welcome → Privacy Notice → Insert Card → Read Card → Face Capture → Confirm Data → Consent Form → Send to Netka EnTelligence → Complete

Returning User Flow

Welcome → Insert Card → Read Card → Show Previous Consent → Edit & Re-submit → Send to Netka EnTelligence → Complete

PDPA Compliance: ระบบรองรับ Granular Consent — แยกความยินยอมเป็นรายข้อ ผู้ใช้เลือกยินยอมหรือไม่ยินยอมแต่ละข้อได้อิสระ พร้อมบันทึก log ครบถ้วน (ใคร, อะไร, เมื่อไหร่, ผ่านช่องทางใด)

2. Actors & Systems

Actor / System Description
Visitor (ผู้ใช้บริการ) ลงทะเบียนและให้ความยินยอมที่หน้าตู้ Kiosk
Kiosk Terminal แสดง UI, สั่ง Card Reader, ถ่ายภาพใบหน้า
Card Reader (Hardware) อ่านข้อมูลจากชิปบัตรประชาชน ผ่าน SignalR Hub
Camera / CCTV (Hardware) ถ่ายภาพใบหน้าสด (Live Photo) พร้อม Face Detection
N-Kiosk Web Server ประมวลผล, จัดเก็บข้อมูล, จัดการ Consent (ASP.NET Core)
File Server (nginx) จัดเก็บไฟล์รูปภาพ (รูปบัตร, รูปใบหน้า)
Netka EnTelligence (External) ระบบปลายทางที่รับข้อมูลลงทะเบียน, ภาพถ่าย, และ Consent
NDPP Server (External) ระบบจัดการ Privacy Consent ส่วนกลาง

3. Preconditions

  • ตู้ Kiosk เปิดอยู่และเชื่อมต่อ Web Server ได้
  • Card Reader เชื่อมต่อผ่าน SignalR Hub สำเร็จ
  • กล้อง / CCTV พร้อมใช้งาน
  • ผู้ใช้มีบัตรประชาชนแบบ Smart Card
  • File Server (nginx) พร้อมรับ upload
  • Netka EnTelligence API พร้อมรับข้อมูล

5. Step Specifications

i18n: ทุกหน้าจอรองรับสองภาษา (ไทย/อังกฤษ) ผ่านระบบ Internationalization — ข้อความบน UI แปลอัตโนมัติตามภาษาที่เลือก โดยมีปุ่มสลับภาษา TH | EN แบบ toggle ที่มุมบนขวาของหน้าจอ (ดูรายละเอียดใน Section 11)

5.0 Login — หน้าเข้าสู่ระบบ

Description: หน้าเข้าสู่ระบบสำหรับผู้ดูแล Kiosk เพื่อเลือก Kiosk ที่จะใช้งาน
UI Elements:
  - Header แสดง "Identity Verification System" บนพื้นหลังสีน้ำเงินเข้ม (gradient)
  - Input fields: Username, Password
  - Checkbox "Remember" สำหรับจดจำข้อมูลล็อกอิน
  - ปุ่ม "Login" สีน้ำเงินเข้ม
Action: กรอก Username/Password → เลือก Kiosk Code → เข้าสู่หน้า Welcome
Storage: บันทึก LoginDetail และ selectedKioskCode ลง localStorage
Navigation: Login สำเร็จ → Welcome (Step1)

5.1 Welcome (Step1) — หน้าจอต้อนรับ

Description: หน้าจอแรกที่ผู้ใช้เห็นเมื่อเดินมาที่ตู้ Kiosk แสดงข้อความ "Verify Identity with National ID Card" พร้อมภาพบัตรประชาชนตัวอย่างและ icon มือแตะ
UI Elements:
  - Top bar: โลโก้ "Netka-ThaiSmart" + เมนู navigation (Steps, Kiosk List, Setup Kiosk, Card Reader & Identity, Logout)
  - ปุ่มสลับภาษา (EN/TH พร้อม flag icon) มุมบนซ้าย
  - ข้อความ "Verify Identity with National ID Card" ตรงกลาง
  - ภาพบัตรประชาชนตัวอย่างขนาดใหญ่ พร้อม icon มือแตะหน้าจอ
  - Footer: "© 2025 Powered by Netka System" + Kiosk Code (KIOSK-0007)
  - พื้นหลัง gradient สีน้ำเงินเข้ม
Action: แตะหน้าจอเพื่อเริ่มกระบวนการยืนยันตัวตน
System: Clear session storage ทั้งหมด เพื่อเริ่มรอบใหม่
Navigation: แตะหน้าจอ → Steps Overview (Step2)
Auto-reset: ถ้า idle เกินเวลาที่กำหนด จะ refresh กลับหน้านี้

5.2 Steps Overview (Step2) — แนะนำขั้นตอน

Description: แสดงภาพรวมขั้นตอนการยืนยันตัวตน มี heading "Steps / Identity Verification" พร้อมแสดงขั้นตอนหลักด้วย circle number icons
UI Elements:
  - Heading: "Steps" (สีอ่อน) + "Identity Verification" (ตัวหนา)
  - ขั้นตอนที่ 1: "Insert your National ID Card into the card reader" (วงกลมเลข 1)
  - ขั้นตอนที่ 4: "Remove your National ID Card from the card reader" (วงกลมเลข 4)
  - พื้นหลัง gradient สีน้ำเงินเข้ม พร้อมลวดลาย wave ด้านล่าง
Action: Auto-advance ไปหน้าถัดไปหลังแสดงครบ
Navigation: auto → Terms and Conditions (Step3)

5.3 Terms and Conditions (Step3) — ข้อตกลงและเงื่อนไข

Description: แสดงข้อตกลงและเงื่อนไขการใช้บริการ (Terms and Conditions) ในกรอบสีขาวบนพื้นหลังสีน้ำเงิน ผู้ใช้ต้องอ่านและยอมรับก่อนดำเนินการต่อ
UI Elements:
  - Heading: "Terms and Conditions" สีน้ำเงินเข้ม
  - กรอบสีขาวแสดงเนื้อหา Terms and Conditions of Use (scrollable)
  - หัวข้อย่อย: "Terms and Conditions of Use", "Reservation of Rights and Limitation of Liability"
  - Checkbox: "I accept the terms and conditions" (ต้องติ๊กก่อนกด Confirm)
  - ปุ่ม "Confirm" สีเทาเข้ม/น้ำเงิน ขนาดใหญ่ มุมโค้งมน
Data Source: เนื้อหา Privacy Notice สามารถดึงจาก NDPP ผ่าน GetPrivacyNotice proxy API
  - เรียก GetPolicyOuByKey ด้วย key จาก variable kiosk_ndpp_privacy_notice
  - แสดง HTML content จาก NDPP / ถ้าไม่มีจะแสดง fallback content เดิม
PDPA Requirement: ต้องแจ้งให้ทราบก่อนเก็บข้อมูล (Privacy Notice)
Action: ติ๊ก checkbox ยอมรับ → กดปุ่ม "Confirm"
Navigation: Confirm → Insert Card (Step4) / ไม่ยอมรับ → ปุ่ม Confirm จะยังกดไม่ได้

5.4 Insert Card (Step4) — สอดบัตรประชาชน

Description: แสดงคำแนะนำการสอดบัตรประชาชน มี step number "1" ในวงกลม พร้อมหัวข้อ "Please insert your National ID Card"
UI Elements:
  - วงกลมหมายเลข "1" ด้านบน
  - ข้อความ "Please insert your National ID Card"
  - ฝั่งซ้าย: animation มือสอดบัตรเข้าช่อง card reader (ช่องสีดำ)
  - ฝั่งขวา: ภาพบัตรประชาชนตัวอย่าง + คำแนะนำ 3 ข้อ
    1. Prepare your National ID Card
    2. Face the card up as shown and insert into the card reader
    3. Do not remove the card until the process is complete
  - Icon บ้าน (home) ด้านล่างสำหรับกลับหน้าแรก
System: SignalR (CardReaderHub) รอ event จาก Card Reader
Timeout: แจ้งเตือนหลัง 5 วินาทีถ้าไม่พบบัตร, auto-reset หลัง timeout
Navigation: พบบัตร → Reading Card (Step5) (auto)

5.5 Reading Card (Step5) — อ่านข้อมูลบัตร

Description: แสดงสถานะกำลังอ่านข้อมูลจากชิปบัตร มี loading animation วงกลมหมุน พร้อมข้อความ "Reading Data" และ "Please Wait..."
UI Elements:
  - Loading spinner animation ด้านบน (วงกลมหมุน)
  - ข้อความ "Reading Data" ขนาดใหญ่
  - ภาพบัตรประชาชนเอียงเล็กน้อย ตรงกลาง
  - ข้อความ "Please Wait..." สี cyan
  - Icon บ้าน (home) ด้านล่าง
Data Read:
  - เลขบัตรประชาชน 13 หลัก
  - ชื่อ-นามสกุล (ไทย/อังกฤษ)
  - วันเกิด, ที่อยู่, เพศ
  - รูปถ่ายจากชิปบัตร
  - วันออกบัตร, วันหมดอายุ
Returning User Check: ระบบตรวจสอบว่าเลขบัตรนี้เคยลงทะเบียนแล้วหรือไม่
Navigation:
  ✓ สำเร็จ → Card Scan Successful (Step8)
  ✗ อ่านไม่ได้ → Card Error (Step6)
  ✗ หมดอายุ → Card Expired (Step7)

5.6 Card Error (Step6) — บัตรอ่านไม่ได้

Trigger: Card Reader อ่านข้อมูลไม่สำเร็จ
Description: แสดงหน้าจอ error พื้นหลัง gradient สีแดง-ส้ม พร้อม icon กากบาทสีขาว หัวข้อ "Please Try Again" และ countdown สำหรับสอดบัตรใหม่
UI Elements:
  - พื้นหลัง gradient แดง-ส้ม
  - Icon กากบาท (X) วงกลมขาว ด้านบน
  - ข้อความ "Please Try Again" ขนาดใหญ่
  - ข้อความ "Reinsert card within ... seconds" (countdown)
  - ภาพบัตรประชาชนตัวอย่าง
  - ข้อความ "Your National ID Card may be damaged or defective"
  - คำอธิบาย 3 ข้อเกี่ยวกับสาเหตุและวิธีแก้ไข
  - ปุ่ม "Cancel Identity Verification" สีขาวตัวแดง มุมโค้งมน
Action: สอดบัตรใหม่ภายในเวลาที่กำหนด หรือ กด "Cancel Identity Verification"
Navigation: สอดบัตรใหม่ → Reading Card (Step5) / Cancel → Welcome (Step1)
Auto-reset: ถ้าไม่กดและหมดเวลา → Welcome (Step1)

5.7 Card Expired (Step7) — บัตรหมดอายุ

Trigger: วันหมดอายุบัตร < วันปัจจุบัน
Description: แสดงหน้าจอ error พื้นหลัง gradient สีแดง-ส้ม พร้อม icon กากบาท หัวข้อ "Unable to Process" และข้อความ "Your National ID Card has expired"
UI Elements:
  - พื้นหลัง gradient แดง-ส้ม (เช่นเดียวกับ Step6)
  - Icon กากบาท (X) วงกลมขาว ด้านบน
  - ข้อความ "Unable to Process" ขนาดใหญ่
  - ภาพบัตรประชาชนตัวอย่าง
  - ข้อความ "Your National ID Card has expired" สีขาว
  - ปุ่ม "Cancel Identity Verification" สีขาวตัวแดง
  - Icon บ้าน (home) ด้านล่าง
Navigation: Cancel → Welcome (Step1)

5.8 Card Scan Successful (Step8) — สแกนบัตรสำเร็จ

Description: แสดงผลสำเร็จการสแกนบัตร พื้นหลัง gradient สีเขียว พร้อม icon ถูกต้อง (checkmark) หัวข้อ "Card Scan Successful"
UI Elements:
  - พื้นหลัง gradient สีเขียว (teal to green)
  - Icon เครื่องหมายถูก (checkmark) วงกลมขาว ด้านบน
  - ข้อความ "Card Scan Successful" ขนาดใหญ่
  - ภาพ animation บัตรสอดอยู่ในช่อง card reader
  - ข้อความ "Please keep your National ID Card inserted until verification is complete"
  - ปุ่ม "Next Step" สีขาวตัวเขียว มุมโค้งมน
Action: กดปุ่ม "Next Step" หรือ auto-advance
Navigation: → Face Scan Guide (Step9)

5.9 Face Scan Guide (Step9) — คำแนะนำสแกนหน้า

Description: แสดงคำแนะนำก่อนสแกนใบหน้า มี step number "2" ในวงกลม หัวข้อ "Face Scan" + "How to Scan Your Face"
UI Elements:
  - วงกลมหมายเลข "2" ด้านบน
  - ข้อความ "Face Scan" ขนาดใหญ่ + subtitle "How to Scan Your Face"
  - คำแนะนำ 2 ข้อ พร้อม icon:
    1. "Please stand straight and look at the camera" (icon กล้อง/face scan)
    2. "If wearing a mask or face covering, please remove it" (icon หน้ากากมีเครื่องหมาย X)
  - พื้นหลัง gradient สีน้ำเงินเข้ม-ม่วง พร้อมลวดลาย wave ด้านล่าง
  - Icon บ้าน (home) ด้านล่าง
Action: Auto-advance ไปหน้าถ่ายภาพ
Navigation: auto → Face Scan (Step10)

5.10 Face Scan (Step10) — สแกนใบหน้า

Description: กล้อง live preview เต็มหน้าจอ พร้อมกรอบตรวจจับใบหน้าและ silhouette แนะนำตำแหน่ง พื้นหลังสีเทาเข้ม/ดำจากภาพกล้อง
UI Elements:
  - ข้อความเตือน (สีแดง) ด้านบน เช่น "ไม่พบใบหน้า กรุณาอยู่ในกรอบกล้อง" เมื่อยังไม่ตรวจพบใบหน้า
  - กรอบสี่เหลี่ยมตรงกลางพร้อม silhouette รูปศีรษะและไหล่ แสดงตำแหน่งที่ควรวางใบหน้า
  - Icon กล้องอยู่ตรงกลาง silhouette
  - พื้นหลังบริเวณนอกกรอบมืดลง (overlay) เพื่อเน้นกรอบ face
  - Icon บ้าน (home) ด้านล่าง
Face Detection: ใช้ face-api.js (TinyFaceDetector) ตรวจจับใบหน้า real-time
Auto-capture: ถ่ายภาพอัตโนมัติเมื่อตรวจพบใบหน้าในกรอบ
Output: Live Photo (base64) → เก็บใน session storage
Navigation: ถ่ายภาพสำเร็จ → Face Scan Successful (Step11)

5.11 Face Scan Successful (Step11) — สแกนหน้าสำเร็จ

Description: แสดงผลสำเร็จการสแกนหน้า พื้นหลัง gradient สีเขียว พร้อม icon ถูกต้อง หัวข้อ "Face Scan Successful" และภาพที่ถ่ายได้
UI Elements:
  - พื้นหลัง gradient สีเขียว (teal to green) เช่นเดียวกับ Step8
  - Icon เครื่องหมายถูก (checkmark) วงกลมขาว ด้านบน
  - ข้อความ "Face Scan Successful" ขนาดใหญ่
  - ภาพใบหน้าที่ถ่ายได้ แสดงตรงกลาง
  - ข้อความ "Please keep your National ID Card inserted until the process is complete"
  - ปุ่ม "Confirm Information" สีขาวตัวเขียว มุมโค้งมน
  - Icon บ้าน (home) ด้านล่าง
Action: กดปุ่ม "Confirm Information" หรือ auto-advance
Navigation: → Data Confirmation (Step12)

5.12 Data Confirmation (Step12) — ตรวจสอบข้อมูล

6.4 API — GetNdppConsented

ดึง consent ล่าสุดของผู้ใช้ เพื่อ prefill checkbox ใน Step17

Item Detail
Method GET
URL /api/KioskApi/GetNdppConsented?citizenId=XXX&kioskCode=XXX
Response { kioskIntegrateNdppId, purposeOptionDetail (JSON string), email, createdDate }

8. Returning User Flow

Requirement: หากผู้ที่เคยลงทะเบียนแล้ว เวลาเสียบบัตรเข้าไป ให้ขึ้นหน้า Consent ที่เคยให้ความยินยอมไว้ แล้วสามารถแก้ไขและส่งกลับไปอีกครั้งได้

8.1 Detection

หลังจากอ่านบัตรสำเร็จ (Step5) ระบบเรียก CheckReturningUser API โดยส่ง เลขบัตรประชาชน (idcard):

API: GET /api/KioskApi/CheckReturningUser?idcard=XXX

Response Field Type Description
isReturning boolean พบเลขบัตรในระบบหรือไม่
consentedId uint ID ของ consent ล่าสุด (เมื่อ isReturning = true)
consentedDate datetime วันที่ consent ล่าสุด

Flow Summary

  • isReturning = true → Returning User Flow:
    1. Step5: Preload NDPP data (GetIntegrateNdpp, GetIntegrateNdppByKiosk, GetNdppPreferenceConsent)
    2. Step8: ข้าม Face Capture (Step9-11) ไปตรง Step17
    3. Step17: Auto-submit consent ทุก service (select all purposes) → NDPP + Netka EnTelligence → StepEnd
  • isReturning = false → New User Flow (Step8 → Face Capture → Confirm → Consent → Complete)
หมายเหตุ: Returning User detection ใช้ เลขบัตรประชาชน (idcard) ตรวจสอบผ่าน API
สำหรับ NDPP ยังมี API GetCheckDsExistConsentForm ที่ใช้ ชื่อ-นามสกุล ผ่าน FindDS() ซึ่งสามารถใช้เสริมได้ในอนาคต

8.2 Returning User — Prefill Consent

Returning User จะเห็น consent form ที่ prefill จากข้อมูลเดิม สามารถแก้ไขก่อน submit ได้:

Step System Behavior
1. ดึง Consent เดิม เรียก GetNdppConsented ดึง purpose + email ที่เคย submit ล่าสุด
2. ดึง Consent Form เรียก GetIntegrateNdppForm สำหรับทุก service
3. Prefill Checkbox purpose ติ๊กตามข้อมูลเดิม + กรอก email เดิมให้อัตโนมัติ (แก้ไขได้)
4. Submit เรียก SubmitIntegrateNdpp → ส่ง consent ไป NDPP + Callback ไป Netka EnTelligence
5. Complete ไป StepEnd

9. Exception Handling

Scenario Detection Handling Navigation
บัตรอ่านไม่ได้ / ชำรุด Card Reader return error แจ้งเตือน → ลองใหม่ หรือ ยกเลิก Retry → Insert Card / Cancel → Welcome
บัตรหมดอายุ expiry_date < today แจ้งเตือน → ยกเลิก Welcome
ไม่สอดบัตรภายใน timeout SignalR timeout auto-reset Welcome
Face detection ล้มเหลว face-api.js no detection ผู้ใช้กดถ่ายใหม่ Face Capture (retry)
File upload ล้มเหลว HTTP non-2xx from File Server Log error, continue flow ดำเนินการต่อ
Netka EnTelligence API ล้มเหลว HTTP non-2xx / timeout Retry + แจ้ง error แสดง error, retry
NDPP API ล้มเหลว HTTP non-2xx Log error + แจ้ง แสดง error
ผู้ใช้ idle idle timer expired auto-reset Welcome
Network disconnect fetch / SignalR failure แสดง error, retry แต่ละหน้า

10. Security & PDPA Compliance

10.1 Security Measures

Measure Description
Payload Encryption ทุก request/response เข้ารหัส AES-ECB ระหว่าง Browser ↔ Web Server
JWT Authentication NDPP consent ใช้ JWT token sign ด้วย SSO_Key
Data Masking เลขบัตรประชาชนแสดงแบบ mask บน UI (unmask ได้ชั่วคราว)
Session Isolation Clear session ทุกครั้งที่เริ่มรอบใหม่
File Server ACL nginx allow/deny จำกัด subnet ที่ upload ได้
HTTPS Production ใช้ HTTPS สำหรับทุกช่องทาง
Citizen ID Encryption เลขบัตรเข้ารหัสก่อนจัดเก็บ
